# 設計哲学の系譜 — なぜこのゲームはこの形なのか

記録日: 2026-07-08(Fable 5 最終日の夜、開発者との対話から)。
これは技術文書ではなく、**このプロジェクトが何であって何でないかを決めた思考の記録**。
半年後の自分と、引き継ぎ先が最初に読むべきもの。

## 座標: 冗談から始まった正確な位置づけ

「ビジネスモデルはVoodoo、クオリティはYostar、目指す方向はアークナイツ系ではなく
Poly Bridge のような考えさせる系統」— 冗談として生まれたこの座標が、
点検すると不自然でなかった。理由は、**収益モデル・生産クオリティ・認知的ジャンル**が
本来は独立した3軸なのに、業界では経済的必然によってセットで固定されてきたから
(磨きは人月を食う→回収装置が要る→ガチャ、等)。AI開発は磨きの限界費用を壊し、
この結合を切断した。「あり得なかった組み合わせ」が成立する時代の、早い一例になる。

## 難しさの材質: 課金で届く世界と届かない世界

- アークナイツの高難度(逆理演算・危機契約・エンドフィールドの論理回路)は
  思考コンテンツだが、難しさの正体は**資源チェック**であり、資源は代替可能
  (腕⇄課金⇄物量に交換レートがある)
- 麻雀・将棋・このゲームの難しさは**対戦相手そのもの**で、何を注いでも代替が効かない
- 中間にポケモン対戦や雀魂(入口の素材は買えるが、ランク戦の結果だけは買えない二層)

このプロジェクトは純粋側に立つ: 収集要素がなく、陣営は対局ごとに選ぶ対称な道具立て。
**課金しても届かない世界**を提供する。ただしそれは「負けの言い訳ができない世界」でも
あるので、CPU戦(腕を作る練習場)を先に整え、非同期PvP(即応の焦りがない)で
梯子を架ける順番が、この渋いラインを歩くための靴になる。

## 3つの失敗からの設計制約

好きだったゲームの失敗を、恨み言ではなく設計制約に変換した。

### アークナイツの失敗 → 時間の主権と可読性

盤面ゲームの思考(全情報を見渡し、比較し、確定するバッチ処理)を
**リアルタイムの器**に流し込んだ(生息演算等)。マップが見えない・追跡しづらい・
物資が速すぎる — UIの出来ではなく「考える時間の所有権がプレイヤーにない」ことの症状。
しかもタワーディフェンス本来の即応の快感まで殺した。

→ 制約1: **非同期ターン制**(思考時間の主権をプレイヤーに返す)
→ 制約2: **見た目がルールを説明する**(可読性は演出ではなく設計原則。
  ジオラマ盤面・地形の文脈ルール・操作性の救済は全部この制約の実装)

### Battle for Wesnoth の失敗 → 軽さと競技インフラ

器(ターン制)は正しいのに時間コストの設計を放棄した。1戦が長く、キャンペーンは
セーブ必須の重さ、同期マルチは相手との時間合わせが必要。競技(Ladder)は
コミュニティ任せで、クライアント信頼ベースのため不正に弱く下火に。
世界観の作り込みに資源が向かい、ゲーム性の方向へ進まなくなった。

→ 制約3: **1戦を軽く**(短い自己完結セッション。Voodooの器とも整合)
→ 制約4: **競技インフラを一級市民に**(権威サーバーで同じ applyAction を検証、
  乱数はサーバー確定、filterStateForViewer で霧の向こうを渡さない —
  Wesnoth Ladder を殺した穴を塞ぐことがアーキテクチャの第一決定だった)
→ 制約5: **世界観はルールに従属**(フィクションパスは表示層のみ。
  「サバゲーフィールドのミニチュア」という世界観自体がルールの説明装置)

### ダンジョンメーカーの失敗 → 運の置き場所

1ターン単位の追跡・毎日の意思決定・やりこみの積み上げは最高だったのに、
ローグライク要素を入れた途端に**運が結果側(ビルドの成立性)にまで侵食**し、
上手さの積み上げを洗い流した。運には「提示の運」(どの選択肢が出るか)と
「結果の運」(選んだ手が機能するか)の2種類があり、失敗の正体は混同にある。

→ 制約6: **運は提示側に閉じ込める**。1戦=完全情報の読める戦術パズル
  (命中のダイスはあるが確率は全て見えている)。将来ローグライク層を載せるなら、
  それは「どのマップ・どの雇用候補・どの強化が提示されるか」だけを握る
  **エンジン外のメタ層**(フィクションパスと同じ「エンジン無傷」の作法)

## ローグライク適性(将来の含み)

Wesnothはキャンペーン分岐を想定した設計だったが中途半端で、1キャンペーンが
重すぎた。このシステムは逆に相性が良い: 短い対局×純関数エンジン
(シード可能・リプレイ可能)×マップ量産の道具(エディタ)が揃っており、
ラン=短い対局の連結+メタ状態、として書ける。
**「1つのエンジン、複数のレンダラー」の原則は「1つのエンジン、複数のゲーム容器」まで
伸びる** — ローグライクのラン(運で遊ぶ日)とガチ対戦の梯子(腕で殴り合う日)が
同じルールの上で共存できる。

## 正直な注記: 起点について

多くのゲームを考察してきたが、このシステムの感情的な起点が
**アークナイツに通ずることは否定できない**(開発者本人の言葉)。
批判した3作は、どれも愛したゲームである。アークナイツは「大衆が高い生産価値の
思考コンテンツを受け入れる」ことを証明し、同時にその方向性を器の制約で
完成させられなかった — この未完への疼きが、別の器でそれを完成させようとする
本プロジェクトの出発点になっている。批判は執着の一形態であり、
このドキュメントの制約群は、好きだったものたちへの返歌でもある。
